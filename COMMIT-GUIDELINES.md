# Dao Quant Research 提交规范

> 规范 Git 提交历史，确保每个目录和文章都有清晰、独立的提交记录

---

## 提交策略

### 核心原则

1. **原子性提交**：每个目录、每篇文章独立提交
2. **详细说明**：提交信息包含内容概要
3. **分类清晰**：按类型（目录/文章/配置）区分提交

---

## 提交信息格式

### 1. 目录结构提交

```
Init: 创建 {分类代码} 目录结构 - {分类名称}

- 分类: {M/I/C/R}{XX} - {分类名称}
- 用途: {说明该分类的用途}
- 预计文章数: {N} 篇
- 子目录: {如有子目录，列出结构}
```

**示例：**
```
Init: 创建 I05 目录结构 - 电子半导体行业研究

- 分类: I05 - 电子半导体
- 用途: 收录电子行业（半导体设备、AI芯片、半导体材料、存储芯片、消费电子）的量化分析文章
- 预计文章数: 10-15 篇
- 子目录: 无（文章直接存放于 I05-semiconductor/）
```

### 2. 文章提交

```
Add: {分类代码}-{序号} {文章标题}

- 分类: {分类代码} - {分类名称}
- 主题: {一句话概括主题}
- 核心内容:
  - {要点1}
  - {要点2}
  - {要点3}
- 难度: {初级/中级/高级}
- 预计阅读时间: {N} 分钟
- 特色: {文章亮点，如模型、数据、案例等}
```

**示例：**
```
Add: I05-01 老子道·芯鉴九维投资分析模型（电子行业细分赛道）

- 分类: I05 - 电子半导体
- 主题: 构建覆盖电子行业五大细分赛道的九维量化投资分析模型
- 核心内容:
  - 九维评估体系：制程节点、国产替代、资本开支、供需缺口、技术路线、下游需求、产业链位置、供应链安全、研发管线
  - 五大细分赛道评分：半导体设备(92)、AI芯片(90)、半导体材料(85)、存储芯片(84)、消费电子(76)
  - 道家哲学映射：道法自然、无为而治、物极必反等投资映射
  - 量化择时信号：各赛道买入/持有/减仓/回避的量化条件
- 难度: 高级
- 预计阅读时间: 45 分钟
- 特色: 以道家哲学为底层逻辑，系统化评估电子行业投资价值
```

### 3. 配置文件提交

```
Config: {文件名称} - {用途说明}

- 文件: {文件路径}
- 用途: {说明用途}
- 关键配置: {列出关键配置项}
```

**示例：**
```
Config: WRITING-GUIDELINES.md - 文章写作规范

- 文件: WRITING-GUIDELINES.md
- 用途: 规范所有研究文章的写作格式、Frontmatter、目录结构
- 关键配置:
  - 4大板块27个子分类体系
  - Frontmatter 必填字段规范
  - 文件命名规则: {分类代码}-{序号}-{kebab-case标题}.md
  - 质量检查清单
```

### 4. README 更新提交

```
Update: README - {更新内容简述}

- 更新类型: {文章列表/目录结构/使用说明/其他}
- 变更内容:
  - {变更1}
  - {变更2}
- 影响范围: {影响的部分}
```

**示例：**
```
Update: README - 添加 I05-01 文章及阅读引导

- 更新类型: 文章列表 + 阅读引导
- 变更内容:
  - 添加最新文章板块，展示 I05-01 芯鉴九维模型
  - 添加阅读指南：按读者类型、投资场景、难度标识分类
  - 更新文章统计：I05 分类文章数 1 篇
- 影响范围: 文章目录、阅读引导
```

## 提交步骤示例

### 新建一个分类目录并添加文章

```bash
# 步骤1: 创建目录结构
cd dao-quant-research
mkdir -p articles/I06-new-energy
git add articles/I06-new-energy/
git commit -m "Init: 创建 I06 目录结构 - 新能源行业研究

- 分类: I06 - 新能源
- 用途: 收录光伏、锂电、储能、新能源车等新能源行业的量化分析文章
- 预计文章数: 8-12 篇
- 子目录: 无"

# 步骤2: 添加文章
cp templates/article-template.md articles/I06-new-energy/I06-01-solar-sector.md
# 编辑文章内容...
git add articles/I06-new-energy/I06-01-solar-sector.md
git commit -m "Add: I06-01 光伏行业量化分析框架

- 分类: I06 - 新能源
- 主题: 构建光伏行业（硅料-硅片-电池-组件-电站）全产业链分析框架
- 核心内容:
  - 光伏产业链各环节盈利能力分析
  - 硅料价格波动对产业链利润分配的影响
  - 技术路线对比：PERC vs TOPCon vs HJT
  - 政策驱动与装机量预测模型
- 难度: 中级
- 预计阅读时间: 30 分钟
- 特色: 产业链利润传导机制分析"

# 步骤3: 更新 README
git add README.md
git commit -m "Update: README - 添加 I06-01 光伏行业分析文章

- 更新类型: 文章列表
- 变更内容:
  - 最新文章板块添加 I06-01 光伏行业分析
  - 更新 I06 分类文章数统计
- 影响范围: 文章目录"

# 步骤4: 推送
git push origin main
```

## 快速参考

### 提交类型前缀

| 前缀 | 用途 | 示例 |
|------|------|------|
| `Init:` | 初始化目录结构 | `Init: 创建 I05 目录结构` |
| `Add:` | 添加新文章 | `Add: I05-01 芯鉴九维模型` |
| `Update:` | 更新现有内容 | `Update: README - 添加文章列表` |
| `Config:` | 配置文件/模板 | `Config: WRITING-GUIDELINES.md` |
| `Fix:` | 修复错误 | `Fix: I05-01 修正评分数据` |
| `Refactor:` | 重构/优化 | `Refactor: I05-01 优化表格格式` |

### 常用命令

```bash
# 查看提交历史
git log --oneline -20

# 查看某次提交的详细信息
git show {commit-hash}

# 查看某文件的提交历史
git log --follow --oneline {file-path}

# 撤销最后一次提交（未推送时）
git reset --soft HEAD~1

# 修改最后一次提交信息
git commit --amend -m "新的提交信息"
```

## 注意事项

1. **不要一次提交多个目录或文章**：保持原子性，便于回滚和追溯
2. **提交信息第一行要简洁**：不超过 50 个字符，清晰说明做了什么
3. **详细说明放在第三行之后**：用空行分隔，详细描述内容
4. **及时推送**：完成一组相关提交后及时推送，避免本地堆积
