## 二、Frontmatter 完整规范

每篇文章顶部必须包含 YAML Frontmatter，格式如下：

```yaml
---
# ========== 基础信息（必填）==========
title: "文章标题：简洁准确，不超过30字"
date: "2025-01-15"                    # 发布日期，YYYY-MM-DD 格式
author: "laozdao"                     # 作者，统一用 laozdao

# ========== 分类标签（必填）==========
category: "M01"                       # 一级+二级分类代码
tags:                                 # 关键词标签，3-8个
  - "双引擎模型"
  - "基本面因子"
  - "盈利能力"
  - "ROE"
  
# ========== 文章状态（必填）==========
status: "published"                   # draft | published | archived | updated

# ========== 版本信息（必填）==========
version: "1.0"                        # 语义化版本

# ========== 摘要（必填）==========
summary: "用100-200字概括本文核心内容、方法和结论。这段文字会显示在文章列表和搜索引擎结果中。"

# ========== 扩展信息（可选）==========
update_date: "2025-01-20"             # 最后更新日期（status=updated时必填）
series: "模型理论系列"                 # 系列名称（系列文章必填）
series_order: 1                       # 系列中的顺序
related:                              # 相关文章链接
  - "M01-02-model-overview"
  - "M02-01-profitability"
featured: true                        # 是否推荐（精选文章）
difficulty: "intermediate"            # beginner | intermediate | advanced
reading_time: 15                      # 预估阅读时间（分钟）
---
```

### 2.1 字段详解

#### category 分类代码

- 格式：`{一级分类}{二级分类}-{序号}`
- 示例：`M01-01` 表示模型理论-模型总览-第1篇
- 序号在同一子分类内递增

#### status 状态

| 状态 | 说明 | 使用场景 |
|------|------|---------|
| `draft` | 草稿 | 正在写作，未正式发布 |
| `published` | 已发布 | 正式发布，可对外分享 |
| `archived` | 已归档 | 内容过时，不再维护 |
| `updated` | 已更新 | 已发布文章的重大更新版本 |

#### difficulty 难度

| 难度 | 目标读者 |
|------|---------|
| `beginner` | 量化投资新手 |
| `intermediate` | 有一定基础的投资者 |
| `advanced` | 专业量化研究者 |

## 五、Markdown 格式规范

### 5.1 标题层级

- 一级标题 `#`：文章主标题（仅一个）
- 二级标题 `##`：主要章节
- 三级标题 `###`：小节
- 四级标题 `####`：子节（尽量少用）

**禁止**：跳级使用标题（如一级直接到三级）

### 5.2 列表规范

**无序列表**：
```markdown
- 项目一
- 项目二
  - 子项目（缩进2空格）
  - 子项目
- 项目三
```

**有序列表**：
```markdown
1. 第一步
2. 第二步
3. 第三步
```

### 5.3 表格规范

- 表头必须加粗
- 对齐方式统一
- 复杂表格可放在附录

```markdown
| 因子名称 | 权重 | 计算方式 | 说明 |
|:---------|:----:|:---------|:-----|
| ROE | 15% | 净利润/净资产 | 盈利能力核心指标 |
| 营收增长率 | 10% | (本期-上期)/上期 | 成长能力指标 |
```

### 5.4 代码与公式

**行内代码**：用 `` `code` ``

**代码块**：
```markdown
```python
# Python代码
import pandas as pd
```
```

**公式**：用 LaTeX 语法
```markdown
$$
ROE = \frac{\text{净利润}}{\text{净资产}} \times 100\%
$$
```

### 5.5 图片规范

- 图片统一放在 `assets/figures/{分类代码}/`
- Markdown 引用使用相对路径
- 必须添加图片说明

```markdown
![双引擎四层模型架构](../assets/figures/M01/model-architecture.png)

**图1**：双引擎四层融合模型架构图
```

### 5.6 链接规范

**内部链接**（同一仓库）：
```markdown
[基本面因子分析](./M02-01-profitability.md)
```

**外部链接**：
```markdown
[Tushare数据平台](https://tushare.pro)
```

## 六、语言风格规范

### 6.1 总体原则

| 原则 | 说明 |
|------|------|
| **准确** | 数据、公式、结论必须准确无误 |
| **简洁** | 避免冗余，一句话一个意思 |
| **客观** | 少用绝对化表述，多用数据和逻辑 |
| **易懂** | 专业术语首次出现需解释 |

### 6.2 禁用词汇

| 禁用 | 替代 |
|------|------|
| "必然"、"肯定"、"绝对" | "大概率"、"统计上显著"、"模型显示" |
| "稳赚"、"必涨"、"翻倍" | "具备上涨潜力"、"评分较高" |
| "我觉得"、"我认为" | "分析表明"、"数据显示" |
| "非常简单"、"显而易见" | 直接陈述事实 |

### 6.3 数字与单位

- 百分比统一用 `%` 符号（如 `15.5%`）
- 大数字用千分位（如 `1,234,567`）
- 货币单位统一为人民币元（特殊情况注明）
- 时间格式：`YYYY-MM-DD`（如 `2025-01-15`）
