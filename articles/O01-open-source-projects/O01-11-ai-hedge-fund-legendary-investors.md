# AI Hedge Fund：用19个传奇投资者Agent构建AI对冲基金

> 如果巴菲特、芒格、林奇、费雪、达蒙达兰围坐一桌讨论一只股票，他们会得出什么结论？AI Hedge Fund用LLM让这个思想实验变成了可运行的代码。

### 1.2 项目定位

AI Hedge Fund 是一个**AI驱动的对冲基金概念验证项目**，通过部署19个LLM驱动的Agent——每个Agent模拟一位传奇投资者的投资哲学——协同分析股票并做出投资决策。

> **核心创意**：不是训练一个"万能AI"来做投资，而是让多个"投资大师AI"各抒己见、辩论交锋，最终由组合经理综合决策。

## 三、19个Agent角色详解

### 3.1 投资大师Agent（13位）

每位Agent模拟一位传奇投资者的核心投资哲学：

| Agent | 投资大师 | 核心哲学 | 投资风格 |
|-------|---------|---------|---------|
| **Warren Buffett** | 沃伦·巴菲特 | 以合理价格买入优秀公司 | 价值+质量 |
| **Charlie Munger** | 查理·芒格 | 以公平价格买入优秀企业 | 质量优先 |
| **Peter Lynch** | 彼得·林奇 | 在日常生活中寻找"十倍股" | 成长+实用 |
| **Phil Fisher** | 菲利普·费雪 | 深度"闲聊"调研成长股 | 成长投资 |
| **Ben Graham** | 本杰明·格雷厄姆 | 只买有安全边际的隐藏宝石 | 深度价值 |
| **Bill Ackman** | 比尔·阿克曼 | 大胆建仓，推动变革 | 激进主义 |
| **Cathie Wood** | 凯瑟琳·伍德 | 相信创新和颠覆的力量 | 颠覆性成长 |
| **Michael Burry** | 迈克尔·伯里 | 寻找被严重低估的深度价值 | 逆向投资 |
| **Mohnish Pabrai** | 莫尼什·帕布莱 | 低风险高回报的Dhandho投资 | 低风险套利 |
| **Nassim Taleb** | 纳西姆·塔勒布 | 关注尾部风险和反脆弱性 | 风险管理 |
| **Stanley Druckenmiller** | 斯坦利·德鲁肯米勒 | 宏观不对称机会+成长潜力 | 宏观趋势 |
| **Aswath Damodaran** | 阿斯沃斯·达蒙达兰 | 故事+数字+纪律性估值 | 估值大师 |
| **Rakesh Jhunjhunwala** | 拉克什·强君胡瓦拉 | 印度"大牛" | 价值+成长 |

### 3.2 分析Agent（4位）

| Agent | 职责 | 输出 |
|-------|------|------|
| **Valuation Agent** | 计算内在价值 | 估值信号（买入/卖出/持有） |
| **Sentiment Agent** | 分析市场情绪 | 情绪信号（乐观/悲观/中性） |
| **Fundamentals Agent** | 分析基本面数据 | 基本面信号（强/中/弱） |
| **Technicals Agent** | 分析技术指标 | 技术信号（看涨/看跌/中性） |

### 3.3 决策层（2位）

| Agent | 职责 | 权限 |
|-------|------|------|
| **Risk Manager** | 计算风险指标，设定仓位限制 | 风险否决权 |
| **Portfolio Manager** | 综合所有信号，做出最终交易决策 | 最终决策权 |

## 四、核心设计理念

### 4.1 投资哲学多样性

AI Hedge Fund 的核心设计理念是**投资哲学的多样性**：

```
价值投资 ────── Graham, Buffett, Munger, Burry, Pabrai
成长投资 ────── Lynch, Fisher, Wood
激进主义 ────── Ackman
宏观趋势 ────── Druckenmiller
风险管理 ────── Taleb
估值分析 ────── Damodaran
```

> **设计洞察**：没有一种投资风格能适应所有市场环境。通过引入多元化的投资视角，系统能够从多个角度评估同一只股票，避免单一视角的盲区。

### 4.2 多层决策机制

```
Layer 1: 分析Agent（量化信号）
  → Valuation / Sentiment / Fundamentals / Technicals
  → 输出结构化信号

Layer 2: 投资大师Agent（定性判断）
  → 13位传奇投资者各自发表观点
  → 基于各自的投资哲学分析

Layer 3: 风险管理（风险约束）
  → Risk Manager评估整体风险
  → 设定仓位限制

Layer 4: 最终决策（综合判断）
  → Portfolio Manager综合所有输入
  → 做出最终交易决策
```

### 4.3 与TradingAgents的对比

| 维度 | AI Hedge Fund | TradingAgents |
|-----|--------------|---------------|
| **Agent角色** | 传奇投资者（巴菲特等） | 专业分析师（基本面/技术等） |
| **设计理念** | 投资哲学多样性 | 交易公司流程 |
| **分析方式** | 定性为主（故事+哲学） | 定量为主（数据+指标） |
| **辩论机制** | 隐含在多角色输出中 | 显式的多空辩论 |
| **市场** | 美股为主 | 美股为主 |
| **LLM提供商** | OpenAI/Groq/Anthropic/Ollama | OpenAI/Google/Anthropic等 |

## 五、安装与使用

### 5.1 安装

```bash
# 克隆仓库
git clone https://github.com/virattt/ai-hedge-fund.git
cd ai-hedge-fund

# 安装Poetry（如未安装）
curl -sSL https://install.python-poetry.org | python3 -

# 安装依赖
poetry install

# 配置API密钥
cp .env.example .env
# 编辑 .env 文件，添加API密钥
```

### 5.2 CLI运行

```bash
# 分析股票（使用OpenAI）
poetry run python src/main.py --ticker AAPL,MSFT,NVDA

# 使用本地LLM（Ollama）
poetry run python src/main.py --ticker AAPL,MSFT,NVDA --ollama

# 指定时间范围
poetry run python src/main.py --ticker AAPL --start-date 2024-01-01 --end-date 2024-03-01
```

### 5.3 回测

```bash
# 运行回测
poetry run python src/backtester.py --ticker AAPL,MSFT,NVDA

# 使用本地LLM回测
poetry run python src/backtester.py --ticker AAPL,MSFT,NVDA --ollama
```

### 5.4 Web界面

项目还提供Web应用界面，提供更友好的可视化操作体验。
