# 模型迭代记录：Dao Quant模型的演进历程

> **一句话摘要**：记录Dao Quant模型从概念到实践的迭代演进历程，分享版本改进、回测验证和经验总结。

---

## 一、引言：模型迭代的必要性

### 1.1 为什么模型需要迭代

金融市场是动态变化的，有效的量化模型需要不断适应市场环境：

| 变化因素 | 影响 | 应对策略 |
|---------|------|---------|
| **市场结构变化** | 投资者结构、交易规则变化 | 调整因子权重 |
| **风格切换** | 价值/成长风格轮动 | 动态配置 |
| **因子失效** | 传统因子超额收益下降 | 开发新因子 |
| **数据质量** | 数据源变化、数据错误 | 数据清洗优化 |
| **模型老化** | 模型预测能力下降 | 重新训练优化 |

### 1.2 迭代的基本原则

```
┌─────────────────────────────────────────────────────────┐
│                   模型迭代基本原则                       │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 数据驱动                                            │
│     所有改进基于回测数据和实盘验证                       │
│                                                         │
│  2. 小步快跑                                            │
│     每次迭代改进一个点，快速验证                         │
│                                                         │
│  3. 风险优先                                            │
│     不追求极致收益，优先控制风险                         │
│                                                         │
│  4. 可解释性                                            │
│     保持模型逻辑清晰，便于理解和维护                     │
│                                                         │
│  5. 版本管理                                            │
│     详细记录每个版本的改动和影响                         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 3.3 v0.9 多因子整合版（2024.12）

#### 3.3.1 版本目标

建立完整的因子检验和筛选机制，优化权重配置。

#### 3.3.2 主要改进

1. **引入因子检验**
   - IC测试
   - 分层回测
   - 相关性分析

2. **因子筛选**
   - 剔除IR < 0.2的因子
   - 剔除高度相关因子
   - 保留15个核心因子

3. **权重优化**
   - 基于IC_IR优化权重
   - 引入风险平价思想

#### 3.3.3 回测表现

| 指标 | v0.5 | v0.9 | 改进 |
|------|------|------|------|
| 年化收益 | 15.8% | 18.2% | +2.4% |
| 夏普比率 | 0.78 | 0.92 | +0.14 |
| 最大回撤 | -32.5% | -28.3% | +4.2% |
| 胜率 | 55% | 58% | +3% |

#### 3.3.4 经验教训

- 因子筛选显著提升了模型质量
- 权重优化对风险调整后收益贡献大
- 但仍缺乏动态调整机制

### 3.5 v1.1 动态权重优化版（2025.06）

#### 3.5.1 版本目标

引入市场环境识别和动态权重调整机制。

#### 3.5.2 主要改进

1. **市场环境识别**
   - 基于指数趋势判断牛熊
   - 基于波动率判断市场状态

2. **动态权重调整**
   - 牛市增加量价权重
   - 熊市增加基本面和风控权重
   - 平滑过渡，避免频繁切换

3. **权重调整规则**

| 市场环境 | 基本面 | 量价 | 风控 |
|---------|--------|------|------|
| 牛市 | 55% | 30% | 15% |
| 震荡 | 60% | 25% | 15% |
| 熊市 | 65% | 15% | 20% |

#### 3.5.3 回测表现

| 指标 | v1.0 | v1.1 | 改进 |
|------|------|------|------|
| 年化收益 | 19.5% | 21.8% | +2.3% |
| 夏普比率 | 1.05 | 1.18 | +0.13 |
| 最大回撤 | -25.6% | -23.2% | +2.4% |
| 胜率 | 61% | 64% | +3% |

#### 3.5.4 分年度改进

| 年份 | v1.0 | v1.1 | 改进 |
|------|------|------|------|
| 2019(牛) | 38.5% | 42.3% | +3.8% |
| 2020(牛) | 42.3% | 45.6% | +3.3% |
| 2021(震荡) | 18.6% | 19.2% | +0.6% |
| 2022(熊) | -8.5% | -5.2% | +3.3% |

#### 3.5.5 经验教训

- 动态权重在牛熊市效果显著
- 震荡市效果不明显
- 需要更精细的市场状态划分

### 3.6 v1.2 风控增强版（2025.09）

#### 3.6.1 版本目标

强化风控机制，引入一票否决和预警系统。

#### 3.6.2 主要改进

1. **一票否决机制**
   - 风控得分<30分：评级最高为C
   - 风控得分<20分：评级最高为D

2. **预警系统**
   - 黄色预警：风控得分下降10分
   - 橙色预警：风控得分下降20分
   - 红色预警：风控得分下降30分

3. **仓位动态调整**
   - 根据风控得分调整建议仓位
   - 高风险标的自动降低仓位

#### 3.6.3 回测表现

| 指标 | v1.1 | v1.2 | 改进 |
|------|------|------|------|
| 年化收益 | 21.8% | 22.5% | +0.7% |
| 夏普比率 | 1.18 | 1.28 | +0.10 |
| 最大回撤 | -23.2% | -19.8% | +3.4% |
| 卡玛比率 | 0.94 | 1.14 | +0.20 |

#### 3.6.4 风控效果

| 场景 | v1.1 | v1.2 | 改进 |
|------|------|------|------|
| 2022年熊市回撤 | -23.2% | -19.8% | -3.4% |
| 极端风险事件 | 无保护 | 触发预警 | 显著改善 |
| 尾部风险 | 暴露大 | 暴露小 | 显著改善 |

#### 3.6.5 经验教训

- 风控增强对夏普比率和最大回撤改善明显
- 收益略有提升（避免了一些高风险标的）
- 预警系统需要配合人工判断

## 五、经验总结与教训

### 5.1 成功经验

| 经验 | 说明 |
|------|------|
| **因子质量优于数量** | 精选15-20个高质量因子优于堆砌50个低质量因子 |
| **风控是底线** | 宁可牺牲收益也要控制回撤 |
| **动态调整有效** | 根据市场环境调整权重显著提升表现 |
| **ML是补充而非替代** | ML模型与传统模型结合效果最佳 |
| **持续监控必要** | 模型需要持续监控和维护 |

### 5.2 失败教训

| 教训 | 说明 |
|------|------|
| **过拟合陷阱** | v0.5版本过度优化历史数据，样本外表现差 |
| **数据挖掘风险** | 曾尝试纯数据挖掘因子，缺乏经济逻辑，很快失效 |
| **忽视交易成本** | 早期版本未充分考虑交易成本，实盘表现差于回测 |
| **单一模型依赖** | 过度依赖ML模型导致可解释性和稳定性问题 |

### 5.3 未来优化方向

| 方向 | 计划 | 优先级 |
|------|------|--------|
| **另类数据** | 引入卫星图像、舆情数据 | 中 |
